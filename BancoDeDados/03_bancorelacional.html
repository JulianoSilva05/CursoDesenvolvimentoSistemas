Bancos Relacionais vs NoSQL

Quando usar cada um?

Curso de Desenvolvimento de Sistemas - SENAI

Duração estimada: 4 horas (2h Teoria + 2h Prática)

O Modelo Relacional (SQL)

Baseado em tabelas, linhas e colunas. Estrutura rígida (esquema).

  • Principal característica: Consistência e Integridade.
  • ACID: Atomicidade, Consistência, Isolamento, Durabilidade.
  • Escalabilidade: Vertical (Servidores mais potentes).
  • Exemplos: MySQL, PostgreSQL, Oracle, SQL Server.

O Modelo NoSQL

"Not Only SQL". Surgiu para lidar com Big Data e flexibilidade.

  • Principal característica: Alta disponibilidade e flexibilidade de esquema.
  • BASE: Basic Availability, Soft state, Eventual consistency.
  • Escalabilidade: Horizontal (Mais servidores baratos em cluster).
  • Exemplos: MongoDB, Redis, Cassandra, Neo4j.

Tipos de NoSQL

  • Documentos: JSON/BSON. Flexível. (MongoDB).
  • Chave-Valor: Simples e rápido. Cache. (Redis).
  • Colunar: Para grandes volumes de análise. (Cassandra).
  • Grafos: Relacionamentos complexos. Redes sociais. (Neo4j).

Teorema CAP

Um sistema distribuído só pode garantir 2 de 3 propriedades:

  • Consistência (C): Todos leem o dado mais recente.
  • Disponibilidade (A): O sistema sempre responde, mesmo com falhas.
  • Partição (P): O sistema continua funcionando se a rede falhar.

SQL foca em CA. NoSQL foca em AP ou CP.

Quando usar?

SQL NoSQL
Dados estruturados e relacionais. Dados não estruturados ou variáveis.
Transações financeiras (ACID). Redes sociais, Logs, IoT, Catálogos.
Consultas complexas (JOINs). Velocidade extrema e escala global.

Desafio de Arquitetura

Parte 1: Escolha da Tecnologia (40 min)

Para cada cenário abaixo, escolha SQL ou NoSQL e o tipo (ex: NoSQL Documento, SQL Relacional). Justifique com base em ACID vs BASE e Escalabilidade.

  1. Um sistema bancário internacional que transfere bilhões entre contas.
  2. Um catálogo de produtos de um marketplace global onde cada vendedor define os atributos do seu produto (ex: uns têm voltagem, outros têm validade, outros têm cor).
  3. O sistema de "Likes" e "Feed" do TikTok, com milhões de escritas por segundo.
  4. Um sistema de gestão de estoque de uma farmácia local.

Parte 2: Teorema CAP (40 min)

Explique por que é impossível ter Consistência, Disponibilidade e Tolerância a Partição ao mesmo tempo em um sistema distribuído. Dê um exemplo de falha.

Parte 3: Pesquisa de Ferramentas (40 min)

Pesquise e descreva brevemente o que são: MongoDB, Redis, Neo4j e Cassandra. Para que cada um é "o melhor"?
