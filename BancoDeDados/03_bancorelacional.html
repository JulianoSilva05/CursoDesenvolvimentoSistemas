Quando usar cada um?

O Modelo Relacional (SQL)

Baseado em tabelas, linhas e colunas. Estrutura rígida (esquema).

  • Principal característica: Consistência e Integridade.
  • ACID: Atomicidade, Consistência, Isolamento, Durabilidade.
  • Escalabilidade: Vertical (Servidores mais potentes).
  • Exemplos: MySQL, PostgreSQL, Oracle, SQL Server.

O Modelo NoSQL

"Not Only SQL". Surgiu para lidar com Big Data e flexibilidade.

  • Principal característica: Alta disponibilidade e flexibilidade de esquema.
  • BASE: Basic Availability, Soft state, Eventual consistency.
  • Escalabilidade: Horizontal (Mais servidores baratos em cluster).
  • Exemplos: MongoDB, Redis, Cassandra, Neo4j.

Tipos de NoSQL

  • Documentos: JSON/BSON. Flexível. (MongoDB).
  • Chave-Valor: Simples e rápido. Cache. (Redis).
  • Colunar: Para grandes volumes de análise. (Cassandra).
  • Grafos: Relacionamentos complexos. Redes sociais. (Neo4j).

Teorema CAP

Um sistema distribuído só pode garantir 2 de 3 propriedades:

  • Consistência (C): Todos leem o dado mais recente.
  • Disponibilidade (A): O sistema sempre responde, mesmo com falhas.
  • Partição (P): O sistema continua funcionando se a rede falhar.

SQL foca em CA. NoSQL foca em AP ou CP.
