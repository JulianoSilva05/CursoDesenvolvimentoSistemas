O Problema dos Dados

Antes dos computadores, como guardávamos informações?

  • Arquivos físicos, pastas, gavetas.
  • Problemas: Espaço físico, deterioração, dificuldade de busca, redundância (cópias desatualizadas).

O Banco de Dados surgiu para resolver a organização, segurança e acesso rápido à informação.

O que é um Banco de Dados?

Um Banco de Dados (BD) é uma coleção organizada de informações estruturadas, armazenadas eletronicamente.

Não confunda o "BD" (os dados em si) com o "SGBD" (o software que gerencia).

Analogia: O BD é como uma biblioteca gigante. O SGBD é o bibliotecário que sabe onde está cada livro e controla quem pode pegar emprestado.

O SGBD

Sistema Gerenciador de Banco de Dados (DBMS em inglês).

Funções principais:

  • Permitir a criação, leitura, atualização e exclusão de dados (CRUD).
  • Controlar acesso (segurança/login).
  • Garantir consistência (evitar dados corrompidos).
  • Backup e recuperação.

Exemplos: MySQL, PostgreSQL, Oracle, SQL Server, MariaDB.

Quem trabalha com BD?

  • DBA (Database Administrator): Responsável pela infraestrutura, desempenho, segurança e backups. O "guardião".
  • Desenvolvedor de Software: Cria as aplicações que consomem e gravam dados no BD. Projeta o esquema lógico.
  • Analista de Dados / Cientista de Dados: Extrai informações e insights dos dados armazenados para tomada de decisão (Business Intelligence).

Breve História dos Modelos

  • Modelo Hierárquico (Anos 60): Estrutura de árvore (pai-filho). Rígido.
  • Modelo de Rede (Anos 70): Grafos complexos. Difícil manutenção.
  • Modelo Relacional (Anos 70/80 - Hoje): Baseado em tabelas e álgebra relacional. O padrão da indústria.
  • NoSQL (Anos 2000+): Foco em flexibilidade e big data (Documentos, Grafos, Chave-Valor).

O Modelo Relacional

Proposto por Edgar F. Codd (IBM) em 1970.

Conceitos Fundamentais:

  • Entidade (Tabela): Algo do mundo real sobre o qual queremos guardar dados (ex: Cliente, Produto).
  • Atributo (Coluna): Uma característica da entidade (ex: Nome do Cliente, Preço do Produto).
  • Tupla (Linha/Registro): Uma instância única da entidade.

Chave Primária (Primary Key - PK)

É o atributo (ou conjunto de atributos) que identifica unicamente cada registro em uma tabela.

  • Não pode se repetir.
  • Não pode ser nulo (vazio).
  • Exemplos: CPF, Matrícula, Código do Produto, ID (auto-incremento).
Reflexão Rápida: Por que o "Nome" de uma pessoa é uma péssima chave primária?

Chave Estrangeira (Foreign Key - FK)

É um atributo que cria um vínculo (relacionamento) entre duas tabelas.

Ela aponta para a Chave Primária de outra tabela.

Tabela PEDIDOS (FK: id_cliente) -----> Tabela CLIENTES (PK: id_cliente)

Isso garante a Integridade Referencial: Não posso criar um pedido para um cliente que não existe!

Cardinalidade (Relacionamentos)

Define "quantos" de um lado se relacionam com "quantos" do outro.

  • 1:1 (Um para Um): Um marido tem uma esposa (legalmente). Raro em modelagem.
  • 1:N (Um para Muitos): Um Cliente faz N Pedidos. Um Departamento tem N Funcionários. (O mais comum).
  • N:N (Muitos para Muitos): Um Aluno cursa N Disciplinas, e uma Disciplina tem N Alunos. (Gera uma tabela intermediária).

Introdução ao SQL

SQL (Structured Query Language) é a linguagem universal dos bancos relacionais.

Dividida em subconjuntos:

  • DDL (Data Definition): CREATE, DROP, ALTER (Define a estrutura).
  • DML (Data Manipulation): INSERT, UPDATE, DELETE, SELECT (Mexe nos dados).
  • DCL (Data Control): GRANT, REVOKE (Permissões).

O Comando SELECT

O comando mais usado. Serve para consultar dados.

SELECT coluna1, coluna2 FROM tabela WHERE condicao ORDER BY coluna1;

Exemplo:

SELECT Nome, Email FROM Alunos WHERE Curso = 'Técnico em DS' ORDER BY Nome ASC;

Manipulando Dados

-- INSERIR INSERT INTO Alunos (Nome, Idade) VALUES ('Carlos', 25); -- ATUALIZAR (Cuidado com o WHERE!) UPDATE Alunos SET Idade = 26 WHERE Nome = 'Carlos'; -- DELETAR (MUITO Cuidado!) DELETE FROM Alunos WHERE Idade < 18;
Atenção: Um UPDATE ou DELETE sem WHERE afeta TODA a tabela!

O Mundo NoSQL

Quando usar NoSQL?

  • Volume de dados massivo (Big Data).
  • Estrutura dos dados muda frequentemente (Schema-less).
  • Alta velocidade de escrita/leitura é crítica.

Tipos: Documentos (MongoDB), Chave-Valor (Redis), Grafos (Neo4j), Colunar (Cassandra).

Tendências em Banco de Dados

  • DBaaS (Database as a Service): AWS RDS, Azure SQL, Google Cloud SQL. Você não gerencia o servidor, apenas usa.
  • NewSQL: Tentativa de unir a escalabilidade do NoSQL com a consistência do SQL (ex: CockroachDB).
  • IA e Dados: Bancos de dados vetoriais para suporte a Machine Learning e LLMs.
