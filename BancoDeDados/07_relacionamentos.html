Revisitando as Chaves

Sem chaves, não há banco relacional.

Primary Key (PK)

Identidade única do registro. CPF, ID, Matrícula.

Regra: Única e Não Nula.

Foreign Key (FK)

O elo de ligação. Aponta para a PK de outra tabela.

Regra: Deve existir na tabela de origem.

Chave Composta: Quando uma única coluna não basta (ex: Dia + Hora + Sala), usamos duas ou mais colunas juntas como PK.

Cardinalidade: A Regra do Jogo

Quantos "lá" se relacionam com quantos "cá"?

1:1 (Um para Um)
Ex: Pessoa <-> CPF.
1:N (Um para Muitos)
Ex: Cliente <-> Pedidos.
N:N (Muitos para Muitos)
Ex: Aluno <-> Disciplina.

Vamos ver como implementar cada um no SQL.

Um para Um (1:1)

Cada registro da tabela A tem apenas um correspondente em B.

Exemplo: Gerente e Departamento (Supondo que um gerente só gerencia 1 depto).

Tabela Gerentes (ID, Nome) Tabela Departamentos (ID, Nome, ID_Gerente UNIQUE)

Regra: A FK pode ficar em qualquer lado, mas deve ter a restrição UNIQUE para garantir que não se repita (ninguém gerencia 2 deptos).

Um para Muitos (1:N)

O relacionamento mais comum do mundo.

Exemplo: Cliente e Pedido.

Tabela Clientes (ID [PK], Nome) | | 1 | / \ N Tabela Pedidos (ID, Data, ID_Cliente [FK])

Regra de Ouro: A FK sempre vai para o lado do "Muitos" (N).

Por que? Porque um pedido só tem 1 cliente (cabe na linha). Um cliente tem N pedidos (não caberia numa célula).

Muitos para Muitos (N:N)

Exemplo: Aluno e Disciplina.

  • Um aluno faz várias matérias.
  • Uma matéria tem vários alunos.

Problema: Se colocar FK em Aluno, ele só pode ter 1 matéria. Se colocar em Disciplina, ela só pode ter 1 aluno.

Solução: Tabela Associativa (Pivot Table)
Criamos uma terceira tabela só para guardar os pares.

A Tabela Associativa na Prática

Tabela Alunos (ID, Nome) | | 1 / \ N Tabela Matriculas (ID_Aluno [FK], ID_Disciplina [FK]) \ / N | | 1 Tabela Disciplinas (ID, Nome)
CREATE TABLE matriculas ( id_aluno INT, id_disciplina INT, PRIMARY KEY (id_aluno, id_disciplina), FOREIGN KEY (id_aluno) REFERENCES alunos(id), FOREIGN KEY (id_disciplina) REFERENCES disciplinas(id) );
