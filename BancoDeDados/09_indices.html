O que é um Índice?

Imagine procurar um tópico num livro de 1000 páginas sem o índice remissivo. Você teria que ler página por página (Full Table Scan).

O índice é uma estrutura de dados auxiliar (geralmente B-Tree) que aponta onde está o dado.

Vantagens e Desvantagens

Vantagens

  • Consultas (SELECT) muito mais rápidas.
  • Ordenação (ORDER BY) instantânea.

Desvantagens

  • Ocupa espaço extra em disco.
  • Deixa a escrita (INSERT, UPDATE, DELETE) mais lenta (pois precisa atualizar o índice também).

Quando criar índices?

✅ Crie em colunas que:

  • São usadas frequentemente no WHERE.
  • São usadas em JOINs (Chaves Estrangeiras).
  • São usadas em ORDER BY.

❌ Evite em colunas que:

  • Têm poucos valores únicos (ex: Gênero M/F, Status Ativo/Inativo).
  • São atualizadas com frequência extrema.

Tipos de Índices

  • PRIMARY: Criado automaticamente na Chave Primária. Único e não nulo.
  • UNIQUE: Garante que não haja valores repetidos (ex: CPF, Email).
  • INDEX (Comum): Apenas acelera a busca. Permite repetidos.
  • FULLTEXT: Para busca em textos longos (como Google).

Criando Índices

-- Criar um índice no CPF do cliente CREATE INDEX idx_cpf ON Clientes(CPF); -- Criar um índice composto (Nome + Sobrenome) CREATE INDEX idx_nome_completo ON Clientes(Nome, Sobrenome); -- Apagar um índice DROP INDEX idx_cpf ON Clientes;
