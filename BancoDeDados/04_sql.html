DML - Data Manipulation Language

São os comandos usados para interagir com os DADOS dentro das tabelas, não com a estrutura (DDL).

CRUD Básico

  • Create -> INSERT
  • Read -> SELECT
  • Update -> UPDATE
  • Delete -> DELETE

Anatomia de uma Tabela

Imagine uma planilha Excel:

Colunas: Campos (Nome, Idade).

Linhas: Registros (João, 25).

SELECT: A Arte de Perguntar

O comando mais usado. Serve para recuperar dados.

-- Traz TODAS as colunas (Cuidado com performance!) SELECT * FROM clientes; -- Traz apenas colunas específicas (Recomendado) SELECT nome, email FROM clientes; -- Renomeando a coluna na saída (Alias) SELECT nome AS 'Nome do Cliente', email FROM clientes;

WHERE: Filtrando Resultados

Sem o WHERE, o banco traz tudo. Use operadores lógicos.

Operadores Básicos

  • = Igual
  • <> Diferente (ou !=)
  • > Maior
  • < Menor
  • >= Maior ou Igual

Exemplos

SELECT * FROM produtos WHERE preco > 100; SELECT * FROM alunos WHERE turma = '1A';

Lógica Booleana: AND, OR, NOT

Combinando múltiplas condições.

-- E (AND): As duas condições devem ser verdadeiras SELECT * FROM funcionarios WHERE setor = 'TI' AND salario > 5000; -- OU (OR): Pelo menos uma deve ser verdadeira SELECT * FROM produtos WHERE cor = 'Azul' OR cor = 'Vermelho'; -- NÃO (NOT): Inverte a condição SELECT * FROM clientes WHERE NOT cidade = 'São Paulo';

INSERT: Adicionando Dados

Insere novas linhas na tabela.

-- Sintaxe Completa (Mais segura) INSERT INTO clientes (id, nome, email) VALUES (1, 'Ana Silva', 'ana@email.com'); -- Inserindo Múltiplos INSERT INTO categorias (nome) VALUES ('Eletrônicos'), ('Livros'), ('Roupas');
Dica: Se a coluna for AUTO_INCREMENT (como ID), você não precisa (e nem deve) passar o valor dela no INSERT.

UPDATE: Atualizando Dados

Modifica registros existentes.

UPDATE clientes SET email = 'novo@email.com', telefone = '9999-0000' WHERE id = 10;
⚠️ ATENÇÃO MÁXIMA!
Se você esquecer o WHERE, vai atualizar TODOS os registros da tabela.
Ex: Todos os clientes passarão a ter o mesmo email. Isso causa demissão!

DELETE: Removendo Dados

Apaga linhas da tabela.

DELETE FROM clientes WHERE id = 5;
⚠️ PERIGO!
DELETE FROM clientes; (sem WHERE) apaga TODOS os clientes.
Sempre faça um SELECT com o mesmo WHERE antes para ter certeza do que está apagando.

LIKE: Buscando Padrões

Quando você não sabe o valor exato.

Wildcards (Curingas)

  • %: Substitui zero ou mais caracteres.
  • _: Substitui EXATAMENTE um caractere.
-- Começa com 'A' WHERE nome LIKE 'A%'; -- Termina com 'Silva' WHERE nome LIKE '%Silva'; -- Contém 'Maria' em qualquer lugar WHERE nome LIKE '%Maria%';

Organizando a Casa

Ninguém gosta de dados bagunçados.

-- ORDER BY: Ordena (Padrão ASC - Crescente) SELECT * FROM produtos ORDER BY preco DESC; -- Do maior para o menor -- LIMIT (MySQL/Postgres) ou TOP (SQL Server): Limita a quantidade SELECT * FROM logs ORDER BY data_hora DESC LIMIT 10; -- Traz apenas os 10 últimos

Lidando com NULL

NULL não é zero, nem string vazia. NULL é "ausência de valor" ou "desconhecido".

Regra de Ouro: Nunca use = NULL. Use IS NULL.

-- ERRADO (Não retorna nada) SELECT * FROM clientes WHERE telefone = NULL; -- CORRETO SELECT * FROM clientes WHERE telefone IS NULL; -- Negativa SELECT * FROM clientes WHERE telefone IS NOT NULL;
