O que é uma Tabela?

A tabela é a estrutura fundamental de um Banco de Dados Relacional.

Ela define o que vamos guardar e como vamos guardar.

Clientes
ID INT
Nome VARCHAR
DataNasc DATE

Antes de inserir dados (DML), precisamos definir a estrutura (DDL).

Tipos de Dados (Data Types)

Cada coluna precisa ter um tipo definido. Os mais comuns são:

  • INT: Números inteiros (ex: Idade, Quantidade).
  • DECIMAL(10,2): Números com vírgula (ex: Preço, Salário).
  • VARCHAR(n): Texto de tamanho variável (ex: Nome, Email).
  • CHAR(n): Texto de tamanho fixo (ex: UF, Sexo 'M'/'F').
  • DATE / DATETIME: Datas e Horas.
  • BOOLEAN: Verdadeiro ou Falso (TinyInt em alguns bancos).

CREATE TABLE

A sintaxe básica é:

CREATE TABLE NomeDaTabela ( NomeColuna1 TipoDeDado [Regras], NomeColuna2 TipoDeDado [Regras], ... );

Exemplo:

CREATE TABLE Produtos ( Codigo INT, Descricao VARCHAR(100), Preco DECIMAL(10,2) );

Constraints (Restrições)

São regras para garantir a qualidade dos dados.

  • PRIMARY KEY (PK): Identificador único (não repete, não nulo).
  • NOT NULL: Obrigatório preencher.
  • UNIQUE: Não pode repetir (mas aceita um nulo, dependendo do banco).
  • DEFAULT: Valor padrão se nada for informado.
  • FOREIGN KEY (FK): Liga com outra tabela.

Exemplo Completo

CREATE TABLE Funcionarios ( ID INT PRIMARY KEY AUTO_INCREMENT, Nome VARCHAR(100) NOT NULL, CPF CHAR(11) UNIQUE NOT NULL, DataAdmissao DATE DEFAULT CURRENT_DATE, Salario DECIMAL(10,2), Ativo BOOLEAN DEFAULT TRUE );
AUTO_INCREMENT: O banco gera o número sozinho (1, 2, 3...).

ALTER TABLE (Alterando Estrutura)

Esqueceu uma coluna? Precisa mudar o tipo? Use o ALTER.

-- Adicionar coluna ALTER TABLE Funcionarios ADD COLUMN Email VARCHAR(100); -- Modificar tipo ALTER TABLE Funcionarios MODIFY COLUMN Salario DECIMAL(12,2); -- Excluir coluna ALTER TABLE Funcionarios DROP COLUMN Ativo;

DROP TABLE (Excluindo Tudo)

Apaga a tabela inteira e todos os dados dentro dela.

DROP TABLE Funcionarios;

CUIDADO! Não tem "Lixeira" (Ctrl+Z) para isso em produção.

Boas Práticas de Modelagem

  • Nomes de tabelas no Plural (Clientes, Pedidos) ou Singular (Cliente, Pedido) - Padronize!
  • Evite acentos e espaços (use snake_case ou CamelCase).
  • Sempre defina uma Primary Key.
  • Use o tipo de dado correto (não use VARCHAR para datas!).

Desafio (Para quem terminou)

Desafio: Sistema de Biblioteca

Modele e crie as tabelas para um sistema de biblioteca:

  • Livros: ISBN (PK), Titulo, Ano, Editora.
  • Categorias: ID, Nome (ex: Ação, Terror).
  • Relacionamento: Adicione uma FK em Livros apontando para Categoria.
  • Empréstimos: Tabela que liga Aluno e Livro, com Data de Empréstimo e Data de Devolução.

Dica: Empréstimos terá duas chaves estrangeiras!

Dicas de Ensino (Turma Heterogênea)

  • Alunos Lentos: Foque na sintaxe básica (CREATE TABLE). Faça eles desenharem a tabela no papel antes de codar.
  • Alunos Rápidos: Peça para pesquisarem sobre CONSTRAINT CHECK (validar se salário > 0) e INDEX.
  • Pair Programming: Coloque um aluno rápido com um mais lento para mentoria.
