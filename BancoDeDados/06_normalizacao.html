Normalização de Dados

Organizando para evitar redundância e anomalias

Curso de Desenvolvimento de Sistemas - SENAI

Duração estimada: 4 horas (2h Teoria + 2h Prática)

O que é Normalização?

É um conjunto de regras aplicadas ao modelo de dados para garantir que ele esteja organizado, sem redundâncias desnecessárias e protegido contra anomalias.

Objetivos:

  • Economizar espaço (não repetir dados).
  • Facilitar a manutenção (alterar em um só lugar).
  • Garantir a integridade dos dados.

Por que normalizar? (Anomalias)

Imagine uma tabela com tudo misturado (Cliente + Endereço + Pedido).

  • Inserção: Não consigo cadastrar um cliente sem ele fazer um pedido.
  • Exclusão: Se apagar o último pedido, perco os dados do cliente.
  • Atualização: Se o cliente mudar de endereço, tenho que alterar em 1000 pedidos (risco de erro).

1FN - Atomicidade

Regras:

  1. Cada coluna deve ter apenas um valor (atômico). Nada de listas separadas por vírgula!
  2. Não deve haver colunas repetidas para o mesmo fim (Tel1, Tel2, Tel3).
  3. Deve haver uma Chave Primária.
Solução para telefones múltiplos: Criar uma tabela separada `Telefones`.

2FN - Dependência Parcial

Regras:

  1. Deve estar na 1FN.
  2. Atributos não-chave devem depender da chave inteira, não de apenas parte dela (em chaves compostas).

Exemplo: Tabela `Notas_Alunos` (ID_Aluno, ID_Materia, Nota, Nome_Materia).

Erro: `Nome_Materia` depende só de `ID_Materia`, não do aluno. Deve ir para a tabela `Materias`.

3FN - Dependência Transitiva

Regras:

  1. Deve estar na 2FN.
  2. Nenhum atributo não-chave deve depender de outro atributo não-chave.

Exemplo: Tabela `Pedidos` (ID, Data, Valor_Total, ID_Cliente, Nome_Cliente).

Erro: `Nome_Cliente` depende de `ID_Cliente` (que não é a PK do pedido). Deve ficar na tabela `Clientes`.

Atividade Prática (2 Horas)

Normalizando uma Nota Fiscal

Considere uma tabela desnormalizada chamada NotaFiscal com os campos:

(Num_NF, Data, Cod_Cliente, Nome_Cliente, End_Cliente, Cod_Prod, Desc_Prod, Qtde, Valor_Unit, Valor_Total_Item)

Parte 1: 1FN (40 min)

Identifique os problemas de atomicidade e repetição. Como resolver?

Parte 2: 2FN (40 min)

Identifique as dependências parciais (o que depende só do Produto? o que depende só da NF?). Separe em tabelas.

Parte 3: 3FN (40 min)

Identifique dependências transitivas (O endereço depende do Cliente, não da NF). Finalize o modelo com as tabelas resultantes.
