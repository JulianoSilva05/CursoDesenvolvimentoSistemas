O Que é um JOIN?

Em bancos relacionais, os dados estão espalhados em várias tabelas para evitar repetição (Normalização).

O JOIN é a cola que junta essas tabelas de volta na hora de consultar.

Tabela A (Clientes)

IDNome
1Ana
2Beto
3Carla

Tabela B (Pedidos)

IDValorID_Cli
100500.001
101200.001
102300.004

Repare: A Carla (ID 3) não comprou nada. O Pedido 102 é do ID 4 (que não existe na tabela Clientes).

INNER JOIN: O Casamento Perfeito

Retorna apenas os registros que têm correspondência em AMBAS as tabelas.

Se não casar, ninguém aparece.

Resultado

Apenas Ana (ID 1) aparece, pois ela tem pedidos.

Beto e Carla (sem pedidos) somem.

Pedido 102 (cliente inexistente) some.

-- Exemplo 1: Simples SELECT c.nome, p.valor FROM clientes c INNER JOIN pedidos p ON c.id = p.id_cliente; -- Exemplo 2: Com Filtro (WHERE) SELECT c.nome, p.valor FROM clientes c INNER JOIN pedidos p ON c.id = p.id_cliente WHERE p.valor > 400;

LEFT JOIN: A Esquerda Manda

Traz TUDO da tabela da esquerda (primeira citada), e tenta casar com a direita.

Se não achar correspondência, preenche com NULL.

Resultado

Ana aparece com seus pedidos.

Beto e Carla aparecem, mas com valor NULL.

Ótimo para relatórios de "Quem NÃO comprou nada".

SELECT c.nome, p.valor FROM clientes c -- Esquerda (Principal) LEFT JOIN pedidos p ON c.id = p.id_cliente;

RIGHT JOIN: A Direita Manda

O contrário do LEFT. Prioriza a tabela da direita.

Raramente usado na prática, pois é confuso de ler. Prefira usar LEFT JOIN trocando a ordem das tabelas.

Resultado

Traz todos os Pedidos.

O pedido 102 (do cliente fantasma ID 4) vai aparecer, com Nome do Cliente = NULL.

SELECT c.nome, p.valor FROM clientes c RIGHT JOIN pedidos p ON c.id = p.id_cliente;

FULL JOIN: União Total

Traz tudo de A e tudo de B. Se casar, junta. Se não casar, preenche com NULL dos dois lados.

Nota: MySQL não tem comando FULL JOIN. Simulamos com UNION.

-- Simulação no MySQL SELECT * FROM clientes LEFT JOIN pedidos ... UNION SELECT * FROM clientes RIGHT JOIN pedidos ...

CROSS JOIN (Produto Cartesiano)

Combina CADA linha de A com CADA linha de B.

Não usa a cláusula ON.

Tabela Cores (2 linhas): Azul, Vermelho

Tabela Tamanhos (3 linhas): P, M, G

Resultado (2 x 3 = 6 linhas):

  • Azul - P
  • Azul - M
  • Azul - G
  • Vermelho - P...
Cuidado! Se você fizer isso com tabelas grandes (1000 x 1000), vai gerar 1 milhão de linhas e travar o banco.
