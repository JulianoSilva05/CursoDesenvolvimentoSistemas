Transações e ACID

Garantindo a consistência dos dados

Curso de Desenvolvimento de Sistemas - SENAI

Duração estimada: 4 horas (2h Teoria + 2h Prática)

O que é uma Transação?

É uma sequência de operações (inserts, updates, deletes) que deve ser tratada como um bloco único.

Regra de Ouro: Ou TUDO acontece com sucesso, ou NADA acontece (All or Nothing).

O Exemplo Clássico: Transferência Bancária

João transfere R$ 100 para Maria.

  1. Subtrair 100 da conta de João.
  2. Somar 100 na conta de Maria.

E se a luz acabar depois do passo 1 e antes do passo 2? O dinheiro some?

Com transações, o passo 1 é desfeito automaticamente!

Propriedades ACID

  • A (Atomicity - Atomicidade): A transação é indivisível. (Tudo ou nada).
  • C (Consistency - Consistência): O banco sai de um estado válido para outro estado válido.
  • I (Isolation - Isolamento): Uma transação não pode interferir na outra enquanto não terminar.
  • D (Durability - Durabilidade): Se confirmou (Commit), está salvo para sempre, mesmo que o servidor pegue fogo.

Comandos de Controle

START TRANSACTION; -- Inicia o bloco UPDATE Contas SET Saldo = Saldo - 100 WHERE ID = 1; UPDATE Contas SET Saldo = Saldo + 100 WHERE ID = 2; -- Se tudo deu certo: COMMIT; -- Salva permanentemente -- Se algo deu errado (antes do Commit): ROLLBACK; -- Desfaz tudo o que foi feito desde o início da transação

Níveis de Isolamento

Controlam como transações simultâneas enxergam os dados umas das outras.

  • Read Uncommitted: Leitura suja (vê dados não salvos). Rápido, mas perigoso.
  • Read Committed: Só lê o que já foi "Commitado". (Padrão na maioria).
  • Repeatable Read: Garante que se você ler a mesma linha 2 vezes, o valor será o mesmo.
  • Serializable: Mais rigoroso. Executa um por um (lento, mas 100% seguro).
