O Que é uma Transação?

Uma transação é um conjunto de operações SQL que deve ser executado como se fosse uma única operação.

Lema: "Um por todos, todos por um". Ou tudo dá certo, ou nada acontece.

Exemplo Clássico: Transferência Bancária
1. Tirar R$ 100 da conta A.
2. Colocar R$ 100 na conta B.

Se o sistema travar entre o passo 1 e 2, o dinheiro some? Não se usarmos Transações!

Controlando a Transação

START TRANSACTION; -- Começa a gravar "a lápis" UPDATE contas SET saldo = saldo - 100 WHERE id = 1; UPDATE contas SET saldo = saldo + 100 WHERE id = 2; -- Se tudo correu bem: COMMIT; -- Grava "a caneta" (Definitivo) -- Se deu erro no meio: ROLLBACK; -- Apaga o que foi feito a lápis e volta como era antes

As 4 Leis: ACID

Para um banco ser confiável, ele deve respeitar o ACID:

A - Atomicidade

Indivisível. Tudo ou nada. Não existe "meia transação".

C - Consistência

O banco não quebra regras (constraints) durante a transação. O saldo não pode ficar negativo se houver regra.

I - Isolamento

O que acontece em Vegas, fica em Vegas. Uma transação não vê os dados "sujos" de outra que ainda não terminou.

D - Durabilidade

Confirmou (Commit)? Está salvo no HD. Pode puxar o cabo da tomada que o dado estará lá na volta.

O Poder do ROLLBACK

Imagine um sistema de E-commerce:

1
Criar Pedido (INSERT) -> OK
2
Baixar Estoque (UPDATE) -> OK
3
Cobrar Cartão -> ERRO: Cartão Recusado!

Solução:

O sistema captura o erro e executa ROLLBACK. O Pedido some e o Estoque volta ao normal, como se nada tivesse acontecido.

O Problema da Concorrência

Dois usuários tentam comprar o ÚLTIMO ingresso do show ao mesmo tempo.

Usuário A

Lê estoque: 1

Compra...

Usuário B

Lê estoque: 1 (Antes do A terminar)

Compra...

Resultado sem Isolamento: Venda Dupla (Estoque -1). O banco precisa bloquear (Lock) o registro para o Usuário B esperar o A terminar.

Níveis de Isolamento

Podemos configurar o quão "paranoico" o banco é.

  • Read Uncommitted: Vale-tudo. Lê dados não salvos (rápido, mas perigoso).
  • Read Committed: Só lê dados confirmados (Padrão).
  • Serializable: Fila indiana. Ninguém faz nada em paralelo (Lento, mas 100% seguro).
