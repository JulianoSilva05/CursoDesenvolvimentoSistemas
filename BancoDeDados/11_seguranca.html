Segurança da Informação (CIA)

Três pilares fundamentais:

Confidencialidade

Só quem tem permissão pode ver. (Sigilo).

Integridade

O dado não pode ser alterado indevidamente (Corrupção).

Disponibilidade

O sistema deve estar no ar quando necessário. (Backup/Redundância).

Gerenciando Usuários (DCL)

Comandos para criar usuários e dar poderes.

Regra de Ouro: Nunca use o usuário root na sua aplicação Java/PHP/Node!
-- 1. Criar Usuário CREATE USER 'vendedor'@'localhost' IDENTIFIED BY 'senha123'; -- 2. Dar Permissão (GRANT) GRANT SELECT, INSERT ON loja.pedidos TO 'vendedor'@'localhost'; -- 3. Tirar Permissão (REVOKE) REVOKE DELETE ON loja.pedidos FROM 'vendedor'@'localhost';

Princípio do Menor Privilégio

Um usuário deve ter APENAS as permissões necessárias para fazer seu trabalho. Nem mais, nem menos.

Cenário Ruim

Usuário 'App' tem permissão de DROP TABLE.

Se um hacker invadir a app, ele apaga o banco.

Cenário Bom

Usuário 'App' só tem SELECT, INSERT, UPDATE.

O hacker não consegue destruir a estrutura.

SQL Injection: A Falha #1

Quando o programador concatena o texto do usuário direto na query.

-- Query no Código (PHP/Java): "SELECT * FROM users WHERE user = '" + input + "'"; -- O Hacker digita no campo de usuário: ' OR '1'='1 -- A Query vira: SELECT * FROM users WHERE user = '' OR '1'='1';

Como '1'='1' é sempre VERDADE, o banco retorna TODOS os usuários e loga o hacker como Admin.

A Vacina: Prepared Statements

Nunca concatene strings! Use parâmetros.

-- Correto (Java/JDBC): String sql = "SELECT * FROM users WHERE user = ?"; stmt = conn.prepareStatement(sql); stmt.setString(1, userInput);

O banco entende que o input é apenas um texto literal, e não um comando executável.

Backup é Vida

Existem dois tipos principais de backup:

  • Dump Lógico (Export): Gera um arquivo .sql com os comandos CREATE e INSERT para recriar tudo. (Lento para restaurar).
  • Backup Físico: Copia os arquivos binários do banco. (Rápido, mas complexo).
Dica: Backup que não foi testado (Restore) não existe.

Atividade 1: Criando Papéis (Roles) (40 min)

O DBA da Escola

Escreva os comandos SQL para criar os usuários e dar as permissões corretas:

  1. Usuário 'diretor': Acesso total (ALL PRIVILEGES) no banco Escola.
  2. Usuário 'professor': Pode ver (SELECT) e lançar notas (INSERT/UPDATE) na tabela Notas, mas NÃO pode deletar nada.
  3. Usuário 'aluno': Só pode ver (SELECT) a tabela Notas.

Atividade 2: Hacker Ético (40 min)

Entendendo a Falha

Imagine um sistema de login com a query:

SELECT * FROM users WHERE email = '$email' AND senha = '$senha';

Se você sabe o email da vítima (admin@empresa.com), o que você digitaria no campo SENHA para logar sem saber a senha real?

Dica: Use o truque do comentário de SQL (-- ou #) para ignorar o resto da query.
