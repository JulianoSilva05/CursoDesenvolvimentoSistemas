Por que programar no Banco?

Até agora, usamos o banco apenas para guardar dados. Mas ele pode fazer muito mais!

Performance

Rodar lógica direto no servidor do banco é mais rápido do que trafegar dados pela rede para o Java/PHP processar.

Segurança e Padronização

Garante que a regra de negócio (ex: cálculo de imposto) seja igual para todos os sistemas que acessam o banco.

Views: Tabelas Virtuais

Uma View é um atalho para uma consulta complexa. Ela não guarda dados, guarda a query.

-- Criando a View CREATE VIEW relatorio_vendas AS SELECT c.nome, p.data, p.valor FROM clientes c JOIN pedidos p ON c.id = p.id_cliente; -- Usando a View (Parece uma tabela normal!) SELECT * FROM relatorio_vendas WHERE valor > 1000;
Vantagem: Simplifica queries gigantes e protege colunas sensíveis (você pode criar uma view que não mostra a senha do usuário).

Stored Procedures

São scripts (funções) salvos no banco que podem receber parâmetros e executar vários comandos em sequência.

DELIMITER // CREATE PROCEDURE fechar_pedido(IN p_id_pedido INT) BEGIN -- 1. Muda status para 'Pago' UPDATE pedidos SET status = 'Pago' WHERE id = p_id_pedido; -- 2. Baixa o estoque (Lógica complexa aqui) UPDATE produtos ... END // DELIMITER ; -- Chamando a Procedure CALL fechar_pedido(1050);

Triggers: Ação e Reação

São códigos disparados automaticamente ANTES (BEFORE) ou DEPOIS (AFTER) de um INSERT, UPDATE ou DELETE.

Exemplo: Auditoria

Sempre que alguém mudar o salário de um funcionário, salve o valor antigo em uma tabela de histórico.

Exemplo: Validação

Antes de inserir, verifique se a idade é maior que 18. Se não, cancele.

Criando um Trigger de Auditoria

CREATE TRIGGER auditoria_salario AFTER UPDATE ON funcionarios FOR EACH ROW BEGIN IF OLD.salario <> NEW.salario THEN INSERT INTO log_salarios (id_func, antigo, novo, data) VALUES (OLD.id, OLD.salario, NEW.salario, NOW()); END IF; END;

OLD: Valor antes do update. NEW: Valor novo.

Functions vs Procedures

O MySQL também tem Functions.

Procedure Function
Executa ações (INSERT, UPDATE). Calcula e RETORNA um valor.
Chamada com CALL. Usada dentro do SELECT.
Pode não retornar nada. Obrigatório retornar algo.
