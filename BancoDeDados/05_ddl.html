O Que é DDL?

DDL (Data Definition Language) é o conjunto de comandos que define a ESTRUTURA (o esqueleto) do banco de dados.

Não lida com dados!

Ao contrário do DML (Insert, Update...), o DDL não mexe no conteúdo (João, Maria), mas sim nos recipientes (Tabela Clientes, Coluna Nome).

Principais Comandos

  • CREATE: Cria objetos.
  • ALTER: Modifica objetos.
  • DROP: Exclui objetos.
  • TRUNCATE: Esvazia objetos.

Tipos de Dados Essenciais

Antes de criar tabelas, precisamos saber o que vamos guardar.

Tipo SQL Descrição Exemplo
INT / INTEGER Números inteiros 10, 500, -5
DECIMAL(10,2) Dinheiro / Exatos 199.90
VARCHAR(N) Texto variável (até N) 'Ana Silva'
DATE / DATETIME Data e Hora '2024-12-31'
BOOLEAN Verdadeiro/Falso TRUE, FALSE

CREATE TABLE: O Começo de Tudo

Sintaxe para criar uma nova tabela.

CREATE TABLE funcionarios ( id INT PRIMARY KEY AUTO_INCREMENT, nome VARCHAR(100) NOT NULL, salario DECIMAL(10,2), data_admissao DATE, ativo BOOLEAN DEFAULT TRUE );
Constraints (Regras):
PRIMARY KEY: Identificador único.
NOT NULL: Obrigatório preencher.
DEFAULT: Valor padrão se ninguém informar nada.

ALTER TABLE: Corrigindo o Projeto

Esqueceu uma coluna? Errou o tipo? O ALTER resolve.

-- Adicionar nova coluna ALTER TABLE funcionarios ADD email VARCHAR(100); -- Modificar coluna existente (Aumentar tamanho) ALTER TABLE funcionarios MODIFY nome VARCHAR(200); -- Renomear coluna (Depende do SGBD, ex: MySQL 8) ALTER TABLE funcionarios RENAME COLUMN salario TO remuneracao; -- Excluir coluna (Cuidado! Perde os dados da coluna) ALTER TABLE funcionarios DROP COLUMN ativo;

DROP vs TRUNCATE: Cuidado Extremo

DROP TABLE

Apaga a tabela INTEIRA. Estrutura e dados somem do mapa.

DROP TABLE funcionarios;

TRUNCATE TABLE

Esvazia a tabela (apaga todos os dados), mas mantém a estrutura pronta para reuso. É mais rápido que DELETE sem where.

TRUNCATE TABLE funcionarios;
⚠️ Não existe "Lixeira" para DROP TABLE. Executou, perdeu.

Relacionando Tabelas (Foreign Key)

Como ligar uma tabela na outra?

CREATE TABLE pedidos ( id INT PRIMARY KEY, valor DECIMAL(10,2), id_cliente INT, -- Definindo a relação: FOREIGN KEY (id_cliente) REFERENCES clientes(id) );

Isso garante a Integridade Referencial: Você não consegue criar um pedido para um cliente que não existe.
