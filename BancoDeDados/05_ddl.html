O que é DDL?

DDL (Data Definition Language) é a parte do SQL usada para criar e modificar a estrutura do banco de dados (esquema).

Principais comandos:

  • CREATE: Cria objetos (tabelas, bancos, views).
  • ALTER: Modifica objetos existentes.
  • DROP: Exclui objetos.
  • TRUNCATE: Limpa todos os dados de uma tabela (mantendo a estrutura).

Tipos de Dados Comuns

Escolher o tipo certo economiza espaço e evita erros.

  • INT / INTEGER: Números inteiros.
  • DECIMAL(10,2) / FLOAT: Números decimais (dinheiro, medidas).
  • VARCHAR(n): Texto de tamanho variável (até n caracteres).
  • CHAR(n): Texto de tamanho fixo.
  • DATE / DATETIME: Datas e horas.
  • BOOLEAN: Verdadeiro ou Falso.

CREATE TABLE

Cria uma nova tabela.

CREATE TABLE Alunos ( ID INT AUTO_INCREMENT PRIMARY KEY, Nome VARCHAR(100) NOT NULL, DataNascimento DATE, Email VARCHAR(100) UNIQUE );

Constraints (Restrições): PRIMARY KEY, NOT NULL, UNIQUE, DEFAULT.

ALTER TABLE

Altera a estrutura de uma tabela existente.

-- Adicionar uma coluna ALTER TABLE Alunos ADD Telefone VARCHAR(20); -- Modificar o tipo de uma coluna ALTER TABLE Alunos MODIFY Nome VARCHAR(150); -- Excluir uma coluna ALTER TABLE Alunos DROP COLUMN DataNascimento;

DROP vs TRUNCATE

-- Exclui a tabela INTEIRA (estrutura + dados) DROP TABLE Alunos; -- Exclui apenas os DADOS (mantém a estrutura zerada) TRUNCATE TABLE Alunos;
Cuidado! O comando DROP não tem "Ctrl+Z". Apagou, já era (a menos que tenha backup).

Definindo Chaves Estrangeiras

CREATE TABLE Pedidos ( ID INT PRIMARY KEY, Valor DECIMAL(10,2), ID_Cliente INT, FOREIGN KEY (ID_Cliente) REFERENCES Clientes(ID) );

Isso impede que um pedido seja criado para um cliente inexistente.
