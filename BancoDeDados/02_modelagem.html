Modelagem de Dados

Conceitual, Lógico e Físico

Curso de Desenvolvimento de Sistemas - SENAI

Duração estimada: 4 horas (2h Teoria + 2h Prática)

O que é Modelagem de Dados?

É o processo de criar uma representação visual e estruturada dos dados de um sistema.

Por que modelar?

  • Facilita o entendimento das regras de negócio.
  • Evita erros de estrutura no futuro.
  • Serve como "planta baixa" para a construção do banco.

Níveis de Abstração

1. Conceitual

Visão de Negócio (Cliente)

Foca no O QUE. Representa entidades e relações sem detalhes técnicos. Independente de software.

2. Lógico

Estrutura (Desenvolvedor)

TB_ALUNO
PK ID_ALUNO (INT)
Nome (VARCHAR)
FK ID_CURSO (INT)

Define COMO. Especifica chaves (PK, FK), tipos de dados e cardinalidade. Próximo do real.

3. Físico

Implementação (DBA)

CREATE TABLE Aluno (
  id INT PRIMARY KEY,
  nome VARCHAR(100)
);

O CÓDIGO REAL. Scripts SQL específicos para o banco escolhido (MySQL, Oracle, etc).

MER e DER

MER (Modelo Entidade-Relacionamento): O conceito teórico.

DER (Diagrama Entidade-Relacionamento): A representação gráfica (desenho).

Componentes principais:

  • Entidade: Retângulo.
  • Atributo: Elipse (ou lista).
  • Relacionamento: Losango (ou linha com cardinais).

Entidades

Representam objetos do mundo real ou conceitos abstratos.

  • Forte: Existe por si só (ex: Cliente, Produto).
  • Fraca: Depende de outra para existir (ex: Dependente depende de Funcionário).
  • Associativa: Surge de um relacionamento N:N (ex: Matrícula).

Atributos

Características das entidades.

  • Simples: Um único valor (ex: Nome).
  • Composto: Subdivisível (ex: Endereço -> Rua, Num, Bairro).
  • Multivalorado: Vários valores para o mesmo campo (ex: Telefones).
  • Derivado: Calculado a partir de outros (ex: Idade, derivado de DataNasc).

Exemplo Prático: Sistema Escolar

Descrição do Modelo:

  • Aluno: Identificado por ID (PK) e Matrícula. Possui Nome e Data de Nascimento.
  • Curso: Identificado por ID. Possui Nome e Carga Horária.
  • Professor: Identificado por ID. Possui Nome e Especialidade.
  • Relacionamentos: Aluno Cursa (N:N) Curso; Professor Ministra (1:N) Curso.

Atividade Prática (2 Horas)

Desafio de Modelagem

Parte 1: Biblioteca (40 min)

Crie o Modelo Conceitual (Entidades e Atributos) para uma biblioteca. Considere: Livros, Autores, Editoras e Empréstimos.

Parte 2: Delivery de Comida (40 min)

Identifique as entidades e relacionamentos para um app tipo iFood. Considere: Restaurante, Prato, Cliente, Pedido e Entregador.

Parte 3: Clínica Médica (40 min)

Modele um sistema para clínica. Considere: Paciente, Médico, Consulta e Exame. Identifique atributos Chave (que identificam unicamente).
