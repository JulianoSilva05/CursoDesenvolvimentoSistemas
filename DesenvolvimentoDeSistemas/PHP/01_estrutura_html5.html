Como Funciona a Web?

O Fluxo Básico

1. Cliente (Browser): Você digita o endereço (URL) no Chrome, Edge, etc.

2. Servidor: Computador conectado 24h na internet que guarda os arquivos do site.

3. Resposta: O servidor envia o código HTML, CSS e JS para o seu navegador montar a página.

Colocando no Ar (Deploy)

Para seu site ser visto pelo mundo, ele precisa sair do seu computador (Localhost) e ir para um Servidor de Hospedagem (Host).

Isso é feito enviando os arquivos via FTP (File Transfer Protocol) ou Git (Versionamento).

A Estrutura Básica Detalhada

Todo arquivo HTML deve seguir essa receita de bolo:

<!-- Linha 1: Declaração do Tipo de Documento -->
<!DOCTYPE html> // Avisa ao navegador: "Sou HTML5 moderno!"

<!-- Linha 2: Raiz do Documento -->
<html lang="pt-br"> // Todo o site fica aqui dentro. Define o idioma para o Google/Tradutor.

<!-- Cabeçalho (Invisível na página, visível pro navegador) -->
<head>
  <meta charset="UTF-8"> // Permite acentos (ç, ã, é). Sem isso, vira "coração" -> "cora��o".
  <meta name="viewport" content="..."> // Essencial para funcionar bem no celular (Zoom correto).
  <title>Nome na Aba</title>
</head>

<!-- Corpo (Visível para o usuário) -->
<body>
  <h1>Título Principal</h1>
  <p>Parágrafo de texto.</p>
</body>
</html>

Exemplo de Página Semântica

Semântica é usar a tag certa para o conteúdo certo. Melhora o SEO (Google) e Acessibilidade.


<body>
  <header> <!-- Cabeçalho do Site -->
    <h1>Meu Blog de Receitas</h1>
    <nav> <!-- Navegação/Menu -->
      <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">Salgados</a></li>
      </ul>
    </nav>
  </header>

  <main> <!-- Conteúdo Principal -->
    <section> <!-- Seção de Artigos -->
      <article>
        <h2>Bolo de Cenoura</h2>
        <p>Receita deliciosa...</p>
        <button>Ler mais</button>
      </article>
    </section>
    
    <aside> <!-- Barra Lateral -->
      <h3>Publicidade</h3>
      <img src="ad.jpg" alt="Propaganda">
    </aside>
  </main>

  <footer> <!-- Rodapé -->
    <p>© 2026 - Todos os direitos reservados</p>
  </footer>
</body>

Tags Explicadas

  • <header>: Topo do site ou de uma seção.
  • <nav>: Área de links de navegação.
  • <main>: O "recheio" principal da página. Só deve ter um por página.
  • <section>: Agrupa conteúdos relacionados.
  • <article>: Conteúdo independente (Notícia, Post, Produto).
  • <aside>: Conteúdo lateral (Relacionado, mas não essencial).
  • <footer>: Rodapé (Copyright, Contato).

O Que é CSS?

Definição

Cascading Style Sheets (Folhas de Estilo em Cascata).

É a linguagem responsável pela apresentação visual do site.

A Analogia da Arara

HTML

A Estrutura (O Esqueleto)

CSS

A Aparência (As Penas)

JS

O Comportamento (A Vida)

Sintaxe Básica

O CSS funciona com regras simples: QUEM eu quero alterar e O QUE eu quero alterar.


/* Seletor { Propriedade: Valor; } */

p {
    color: red;          /* Cor do texto */
    font-size: 16px;     /* Tamanho da fonte */
    text-align: center;  /* Alinhamento */
}

Seletores: Class vs ID

Class (Classe) .nome-da-classe

Símbolo: Ponto (.)

Uso: Reutilizável. Pode ser usado em VÁRIOS elementos na mesma página.

Exemplo: Botões, cards, textos destacados.


HTML: <button class="btn-azul">Salvar</button>
      <button class="btn-azul">Enviar</button>

CSS:  .btn-azul { background-color: blue; }

ID (Identificador) #nome-do-id

Símbolo: Cerquilha/Hashtag (#)

Uso: Único. Só pode existir UM elemento com esse ID na página.

Exemplo: Cabeçalho principal, Rodapé, Menu lateral, Formulário de Login.


HTML: <header id="topo-site">...</header>

CSS:  #topo-site { height: 100px; }

Dica de Ouro: Na dúvida, use Class. Use ID apenas para elementos estruturais únicos ou para manipulação via JavaScript (ex: getElementById).

3 Formas de usar CSS

1. Inline (Na linha) (NÃO RECOMENDADO)

<p style="color: blue; font-size: 20px;">Texto</p>

Por que evitar? Difícil manutenção. Se quiser mudar a cor de todos os parágrafos, terá que editar um por um. Suja o HTML.

2. Incorporado (Interno) (MÉDIO)

Usa a tag <style> dentro do <head>.

Uso: Bom para páginas únicas (Landing Pages simples) ou e-mails marketing. Ruim para sites grandes (não compartilha estilo entre páginas).

3. Externo (Arquivo .css) (RECOMENDADO)

<link rel="stylesheet" href="style.css">

Vantagem: Separação total (HTML limpo). Um único arquivo controla o visual do site inteiro. O navegador salva em cache (carrega mais rápido na segunda visita).

Melhores Práticas (Boas Maneiras)

Organização e Clareza

  • Indentação: Mantenha o código alinhado (filho dentro do pai). Facilita a leitura e evita erros de fechamento de tags.
  • Comentários: Use <!-- Explicação --> para documentar partes complexas. Ajuda seu "eu do futuro".
  • Nomes em Inglês (Opcional mas Bom): Padrão mundial (ex: class="header-container" em vez de "topo-caixa").

Técnicas

  • Semântica Sempre: Evite "Divite" (Uso excessivo de div). Use <button> para botões, não <div onclick>.
  • Separation of Concerns: HTML é Estrutura. CSS é Estilo. JS é Comportamento. Não misture!
  • Acessibilidade: Sempre use o atributo alt="descrição" em imagens para cegos e SEO.

Box Model: O Conceito Fundamental

Todo elemento HTML é uma caixa retangular.


.caixa {
  width: 200px;      /* Largura do conteúdo */
  padding: 20px;     /* Espaço interno (engorda a caixa) */
  border: 5px solid; /* Borda (engorda a caixa) */
  margin: 30px;      /* Espaço externo (empurra os vizinhos) */
}
