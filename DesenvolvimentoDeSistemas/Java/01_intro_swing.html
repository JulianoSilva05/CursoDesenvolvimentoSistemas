Teoria e Conceitos Fundamentais

O que é Swing?

Definição

O Swing é uma biblioteca gráfica (GUI toolkit) para Java, parte do JFC (Java Foundation Classes).

Ele permite criar janelas, botões, tabelas e interfaces ricas que funcionam em qualquer sistema operacional (Windows, Linux, Mac).

Swing vs AWT

  • AWT (Abstract Window Toolkit): A primeira biblioteca do Java. Usava componentes do sistema operacional (peso-pesado). Era limitado e feio.
  • Swing: Escrito 100% em Java (peso-leve). Desenha seus próprios componentes, garantindo que o visual seja o mesmo em todas as plataformas.

Arquitetura MVC no Swing

O Swing utiliza uma variação do padrão Model-View-Controller.

Model (Modelo)

Guarda os DADOS.

Ex: O texto de um botão, os itens de uma lista.

↔

View (Visão)

Mostra na TELA.

Ex: O desenho do botão, a cor, o tamanho.

↔

Controller (Controle)

Gerencia EVENTOS.

Ex: O clique do mouse, digitar no teclado.

No Swing, View e Controller muitas vezes são combinados no componente (Delegate).

Pacotes e Classes Principais

Para programar em Swing, você vai importar muito esses pacotes:

import javax.swing.*; // O principal. Tem JButton, JFrame, etc.
import java.awt.*; // Tem Layouts, Cores, Fontes.
import java.awt.event.*; // Para lidar com cliques e ações.

A Hierarquia Básica

  • java.awt.Component: O avô de todos.
  • java.awt.Container: Pode conter outros componentes.
  • javax.swing.JComponent: A base de quase tudo no Swing. Suporta bordas, tooltips, atalhos.

Recursos Poderosos (Features)

Pluggable Look and Feel (PLAF)

Você pode mudar a "cara" da sua aplicação com uma linha de código.

Pode parecer Windows, Mac, Linux ou ter um visual próprio (Nimbus, FlatLaf).

Acessibilidade

Suporte nativo para leitores de tela e tecnologias assistivas.

Componentes Ricos

Tabelas editáveis, Árvores de diretórios, Editores de texto com estilo (HTML/RTF).

Double Buffering

Desenho suave sem "piscar" a tela (flicker-free).
