Fundamentos de GUI e Arquitetura MVC

Conceitos Fundamentais (Teoria)

Swing vs AWT

Swing é 'Peso-Leve' (desenhado pelo Java), AWT é 'Peso-Pesado' (do Sistema Operacional).

Arquitetura MVC

Separação entre Modelo (Dados), Visão (Tela) e Controle (Eventos).

JComponent

A classe base para todos os componentes visuais do Swing.

Independência

A interface parece a mesma em Windows, Linux e Mac (Look and Feel).

Aprofundamento Técnico

1. A Thread de Eventos (EDT)

O Swing não é Thread-Safe. Todas as atualizações de tela devem ser feitas na Event Dispatch Thread.

SwingUtilities.invokeLater(() -> { ... });

2. Double Buffering

O Swing usa buffer duplo nativamente para evitar que a tela 'pisque' (flicker) ao ser redesenhada.

Exemplos de Implementação

Analise o código abaixo para entender a aplicação:

// Hello World em Swing import javax.swing.*; public class HelloSwing { public static void main(String[] args) { // Garante execução na Thread correta SwingUtilities.invokeLater(() -> { JOptionPane.showMessageDialog(null, "Olá Mundo Swing!"); }); } }
